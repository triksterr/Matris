1) Назначение
- Файл хранит только задачи, их статусы, приоритеты и результаты проверок.
- Формат строк фиксированный и одинаковый для всех записей.
- Свободные заметки и длинные обсуждения сюда не писать.

2) Словарь статусов
- TODO: задача запланирована, работа не начата.
- IN_PROGRESS: задача в работе.
- BLOCKED: задача заблокирована внешним условием/решением.
- DONE: задача завершена и проверена.
- DEFERRED: задача отложена на более поздний этап.
- CANCELLED: задача отменена.

3) Словарь приоритетов
- P0: первостепенная задача текущего этапа.
- P1: важная задача, следующая после P0.
- P2: отложенная задача этапа.

4) Формат строки задачи (строго одна строка)
[ID] | [STATUS] | [PRIORITY] | [STAGE] | [AREA] | [OWNER] | [SUMMARY] | [COMMENT]

Пояснение полей:
- ID: TASK-XXXX (например TASK-0001).
- STATUS: только из словаря статусов выше.
- PRIORITY: только P0/P1/P2.
- STAGE: текущий этап (например WIN_CONSOLE_V1).
- AREA: модуль/подсистема или конкретный диапазон строк файла.
- OWNER: ответственный (USER или CODEX).
- SUMMARY: краткая формулировка задачи (одна строка).
- COMMENT: краткий комментарий в одну строку или `[]`, если комментария нет.

5) Формат строки результата (строго одна строка)
[ID] | [DATE] | [RESULT] | [CHECK] | [ARTIFACTS] | [NEXT] | [COMMENT]

Пояснение полей:
- ID: тот же TASK-XXXX, к которому относится результат.
- DATE: YYYY-MM-DD HH:MM.
- RESULT: кратко что сделано/что выявлено.
- CHECK: как проверено (сборка/запуск/smoke/gtests и т.д.).
- ARTIFACTS: файлы/модули/конфигурации, которые затронуты.
- NEXT: следующий шаг или `-`, если не требуется.
- COMMENT: краткий комментарий в одну строку или `[]`.

6) Формат многострочного COMMENT (для ссылок и длинных пояснений)
[TASK-XXXX] | [COMMENT_CONT] | [текст]

Правила для COMMENT_CONT:
- Используется только после основной строки задачи/результата с тем же ID.
- Одна строка `COMMENT_CONT` = один логический фрагмент.
- Для ссылок плагина: только одна ссылка ` Link:... ` на строку.
- Если нужен только набор ссылок, поле COMMENT в основной строке ставить `[]`.

7) Правила ведения
- Каждую новую задачу добавлять отдельной строкой в формате задачи.
- Изменение статуса отражать новой строкой результата, а не перезаписью истории.
- Для статуса DONE обязательно заполнять CHECK и ARTIFACTS.
- Для статуса BLOCKED обязательно указывать причину в RESULT и действие в NEXT.
- Временная зона дат: Europe/Moscow.
- ID задач уникальны; повтор ID для разных задач запрещен.
