### **4. Техническое проектирование приложения**

#### **Архитектура (Многослойная)**
Чтобы проект был кроссплатформенным, необходимо разделить код на независимые модули.

1.  **Ядро (Core Game Engine)**
    *   **Язык:** C++ 17.
    *   **Независимость:** Полное отсутствие платформо-зависимых вызовов (никаких `Windows.h`, `UIKit` и т.д.).
    *   **Что содержит:**
        *   `GameLogic`: Основная игровая механика, расчет падения, проверка связей, отталкиваний, сбор молекул.
        *   `GameField`: Представление игрового поля (двумерный массив или граф объектов).
        *   `Block`, `Particle`: Классы, описывающие падающие фигуры и отдельные блоки (с цветом, позицией, состоянием).
        *   `Physics`: Модуль, который применяет правила взаимодействия между блоками.
        *   `GameState`: Текущее состояние игры (счет, уровень, цель, игра/пауза/конец).

2.  **Слой платформенной абстракции (Platform Abstraction Layer - PAL)**
    *   **Задача:** Скрыть различия между ОС. Этот слой предоставляет единый интерфейс для ядра для выполнения платформо-зависимых задач.
    *   **Интерфейсы (абстрактные классы в C++):**
        *   `IRenderer`: Отрисовка (например, методы `DrawRect`, `DrawText`).
        *   `IAudio`: Воспроизведение звуков.
        *   `IInput`: Обработка ввода (касания, клавиатура, мышь).
        *   `IFileSystem`: Работа с файлами (загрузка уровней, сохранение прогресса).

3.  **Платформенная реализация (Platform Implementation)**
    *   Для каждой платформы мы создаем реализацию интерфейсов PAL.
    *   **Windows/Linux:** Используем **SFML** или **SDL2**. Эти библиотеки предоставляют реализацию для рендеринга, аудио и ввода. Наш код для этих платформ будет очень легким — просто запуск игрового цикла и вызовы SFML/SDL через PAL.
    *   **Android:**
        *   **Нативный код (Native Activity):** Пишем реализацию `IRenderer` через **OpenGL ES**, `IInput` через обработку событий сенсорного экрана от Android NDK, `IAudio` через **OpenSL ES**.
        *   **Проще:** Можно использовать фреймворк, который уже предоставляет кроссплатформенные возможности, например, **SDL2** (он отлично работает на Android, что сильно упростит портирование).

4.  **Слой пользовательского интерфейса (UI)**
    *   Меню, кнопки "Пауза", "Рестарт", отображение счета и цели.
    *   Для десктопа это могут быть элементы, отрисованные через тот же SFML/SDL.
    *   Для Android можно использовать стандартные View поверх Native Surface или, опять же, отрисовывать все вручную через OpenGL для единообразия.

### **5. Технологический стек**

*   **Язык:** C++ (ядро), Java/Kotlin (для обертки на Android, если не использовать чистый NativeActivity).
*   **Кроссплатформенные библиотеки:** **SDL2** (предпочтительно) или **SFML**. Они сильно упростят портирование.
*   **Графика:** OpenGL ES 2.0/3.0 (через SDL2/SFML) для гарантии работы на мобильных устройствах.
*   **Аудио:** Использовать возможности SDL2_mixer или SFML-аудио.
*   **Сборка:** **CMake** для управления проектом. Он отлично поддерживается Android NDK и всеми основными IDE.
*   **IDE:** Visual Studio / CLion (для десктопа), Android Studio (для сборки Android-пакета).

### **6. План разработки (по этапам)**

1.  **Прототип на C++ для ПК:**
    *   Создать консольную версию для отладки игровой логики (где блоки — это буквы/цифры).
    *   Реализовать ядро: поле, фигуры, базовые правила связывания/отталкивания.

2.  **Визуализация для ПК:**
    *   Подключить SDL2 или SFML.
    *   Реализовать отрисовку поля, цветных блоков, падающих фигур.
    *   Добавить управление с клавиатуры.

3.  **Полировка геймплея:**
    *   Добавить систему уровней и целей ("молекул").
    *   Реализовать подсчет очков.
    *   Добавить звуковые эффекты и музыку.

4.  **Портирование на Android:**
    *   Настроить CMake для сборки под Android через NDK.
    *   Создать простую Android Activity, которая запускает наш нативный код.
    *   Адаптировать управление для сенсорного экрана (тапы для движения, свайпы для поворота).
    *   Протестировать на реальном устройстве.

5.  **Финальные штрихи:**
    *   Добавить меню, настройки.
    *   Проработать графический дизайн (скины для блоков, фон).
    *   Оптимизация производительности для мобильных устройств.
